## Editorial Principles

### 1. Landscape before EA

先建立领域全景，再选定主文献。  
避免先入为主导致遗漏更高质量证据。

### 2. Evidence Gap ≠ Editorial Choice

如果现有最佳文献不完全符合 Coverage 定义，应将 Coverage 状态标注为 **Partial / Contextual**，而不是强行归并或降低标准。

### 3. Version Freeze

v1.0 冻结后不回溯修改结构。  
通过 Release Note 和后续版本演进追踪变化。  
知识库的稳定性比完整性更优先。

### 4. Editorial Role ≠ Evidence Level

Evidence Level 描述研究设计。  
Editorial Role 描述文献在知识体系中的职责。  
同一 Evidence Level 的文献可以承担不同 Editorial Role；  
Editorial Role 由知识图谱中的位置决定，而非仅由方法学等级决定。

## Design Observation

Parent Stress Cluster 的建设过程中，一个模式不断重复：

> 每次决定“先停一步，建立标准”，最终都节省了后期返工成本。

这一模式体现在多个层级：

- 先冻结 Cluster Definition，再归档 EA。
- 先完成 Mechanism Landscape，再确定 EA。
- 先冻结 EA Index，再扩展 Archive。
- 先完成 Trajectory Landscape，再确定 Coverage 定位。
- 先建立规范，再扩展模板。

因此，**Landscape before EA** 并不仅仅是一条文献筛选原则，而是一条贯穿 Repository 建设全过程的方法论原则。
