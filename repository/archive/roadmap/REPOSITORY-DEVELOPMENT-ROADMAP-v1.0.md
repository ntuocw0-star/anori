# Repository Development Roadmap v1.0

**Document Type:** Strategic Reference
**Version:** 1.0
**Established:** 2026-07-13 · Sprint 24 Close · Sprint 25 Step 0
**Purpose:** Every Sprint should know which Stage it is in.

---

## Roadmap Overview

```
Stage I    Knowledge Organization     ✓ Completed   Sprint 18–21
Stage II   Knowledge Integration      ✓ Completed   Sprint 22
Stage III  Knowledge Evolution        ✓ Completed   Sprint 23–24
────────────────────────────────────────────────────────────────
Stage IV   Knowledge Network          ← Current     Sprint 25+
Stage V    Analytics Automation       Future
Stage VI   Repository Intelligence    Long-term
```

---

## Stage I — Knowledge Organization
**Status:** ✓ Completed (Sprint 18–21)

**Goal:** Build the Repository.

**Validated Capabilities:**
- Evidence Article (EA) structure
- Evidence Topic (ET) structure
- Knowledge Article (KA) structure
- Series structure
- Template v1.0–v1.2
- First full Module (Participation)

**Marker:** Repository can organize knowledge.

---

## Stage II — Knowledge Integration
**Status:** ✓ Completed (Sprint 22)

**Goal:** Repository supports Cross-concept connections.

**Validated Capabilities:**
- Cross-concept EA
- Cross-concept ET (first draft structure)
- Shared Evidence Pool concept
- Cross-concept tagging

**Marker:** Repository can integrate knowledge across Concepts.

---

## Stage III — Knowledge Evolution
**Status:** ✓ Completed (Sprint 23–24)

**Goal:** Repository can manage knowledge maturity.

**Validated Capabilities:**
- Template v1.3 (17 rules, frozen)
- Evidence Upgrade Event mechanism
- Repository Status lifecycle: Seed Evidence → Conditionally Ready → Full Evidence
- Shared Dataset Pattern (Validated)
- Architecture Notes Maturity 4-tier system
- Gap Classification System (Repository Coverage Gap vs Field-level Evidence Gap)
- Cross-concept ET with Relationship Direction node
- Evolution Log

**Marker:** Repository can evolve knowledge, not just store it.

---

## Stage IV — Knowledge Network
**Status:** Current (Sprint 25+)

**Goal:** Mature Concepts form a connected knowledge network.

**Target:**

```
Family Support
      │
      ▼
Participation ◄──── Parenting Stress
      ▲
      │
   FQOL ──── Family Functioning
```

**Validation Targets:**
- Cross-concept ET × 3–4 (Participation as first Hub)
- Relationship Direction established for each connection
- Evidence Level tracked per connection
- Gap Classification applied consistently

**Milestone:** Cross-concept Knowledge Graph (when 3–4 connections established)

**Freeze Conditions for Stage IV completion:**
- Participation linked to ≥3 mature Concepts
- Each link has validated Relationship Direction or confirmed Gap
- Knowledge Graph structure established

---

## Stage V — Analytics Automation
**Status:** Future

**Goal:** Repository auto-generates structural views.

**Target Outputs (auto-generated):**
- Coverage Matrix
- Framework Map
- Evidence Pyramid per Concept
- Lifecycle status dashboard
- Repository Status summary
- Knowledge Network visualization

**Prerequisite:** Stage IV Knowledge Graph stable.

---

## Stage VI — Repository Intelligence
**Status:** Long-term

**Goal:** Repository actively identifies what it needs.

**Target Capabilities:**
- Knowledge Gap detection
- Evidence Upgrade Event prediction
- Potential Cross-concept Link suggestion
- Future Research Priority ranking

**Prerequisite:** Stage V automation operational.

---

## Sprint Positioning Guide

Before starting any Sprint, confirm:

```
1. Which Stage is this Sprint in?
2. What capability does this Sprint validate?
3. Does this Sprint advance the Stage, or maintain within it?
4. What is the Freeze Condition for moving to the next Stage?
```
