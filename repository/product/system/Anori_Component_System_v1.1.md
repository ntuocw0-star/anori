## 组件详细规格（C01–C28 保持不变，以下为新增）

---

### C29 Core Box

**用途**: 路径核心原则卡，放在页面开头，建立最重要的认知

**规则**:
- 蓝色背景（bg-accent）+ 蓝色边框
- 字体：16px，font-weight 500，color: text-primary
- 行高 1.7
- 只放一句话或两句话——不能太长
- 适用：每条路径最核心的认知重构，通常在 U001 使用

**模板**:
```html
<div class="core-box">
  <p>[路径核心原则，1–2句，不能超过3行]</p>
</div>
```

**P003 示例**:
```
挑战性行为不是随机的，也不是故意的。它有功能——
是孩子目前能用的最有效方式，用来得到某样东西，
或者逃开某件事。行为是信息，不是问题。
```

---

### C30 Function List

**用途**: 展示行为/情绪/能力的多种功能或来源类型

**规则**:
- 每项：Emoji + 功能名（加粗）+ 说明 + 生活例子（斜体）+ 可选链接
- Anori Functional Language 统一使用（见下方）
- 通常 4–6 项
- 结尾句：「知道卡在哪里，才知道从哪里帮。」

**Anori Functional Language（全站统一，不重新解释）**:
```
🗣️ 我要说什么（沟通需求）
👁️ 看看我（寻求关注）
🚪 我想离开（逃避）
🌀 身体需要（寻求感觉）
😰 我承受不了（情绪/内在状态）
🎁 我要那个（取得实物）
```

---

### C31 Same Behavior Table

**用途**: 展示同一个行为可以来自不同来源

**规则**:
- 表格或列表形式
- 左列：行为（固定同一个，重复显示）
- 右列：对应来源，每行不同
- 结尾说明：「处理行为，如果不知道来源，就像在黑暗里走路。」
- 适用：打破「同一行为=同一原因」的误解

---

### C32 ABC Cards

**用途**: A-B-C 行为观察框架，三色卡

**规则**:
- A卡：蓝色（bg-accent），「发生之前」
- B卡：红色（bg-danger），「发生了什么」
- C卡：绿色（bg-success），「后来发生什么」
- 每卡：字母标识（大字）+ 标题 + 说明 + 自问问题（斜体）
- C卡特别标注：「C 是最关键的线索——它揭示行为的功能」

**ABC Framework 标准命名（全站统一）**:
```
🟦 A — 发生之前（情境、要求、环境）
🟥 B — 发生了什么（事实描述，不加解释）
🟩 C — 后来发生什么（孩子得到了什么或避开了什么）
```

---

### C33 Function Signal List

**用途**: 六种来源各自的观察信号和判断问题

**规则**:
- 每项：Emoji + 功能名 + 信号标签（pill 样式）+ 判断问题（斜体）
- 信号标签：小圆角，灰色边框，font-size 12px
- 通常 6 项，对应 Anori Functional Language
- 适用：帮家长对照自己观察到的信号，判断来源

---

### C34 ABC Record Box

**用途**: 一次 ABC 观察的记录格式

**规则**:
- 灰色背景（surface-1）
- 标签：「一次观察记录（可以直接用这个格式）」
- 四行：A / B / C / 猜（最可能的来源）
- 字体 14px，左列标签加粗
- 适用：给家长一个可以直接使用的记录工具

---

### C35 Function Response List

**用途**: 六种来源对应的当下回应方式

**规则**:
- 每项：Emoji + 功能名 + 回应方式 + 生活例子（斜体）+ 「不要做」（红色小字）
- 「不要做」说明为什么这样做会强化行为，而不只是说不能做
- 通常 6 项，对应 Anori Functional Language
- 适用：行为发生时的快速参考

---

### C36 Replacement Triangle

**用途**: 替代行为的三个必要条件

**规则**:
- 三卡等宽，蓝色背景（bg-accent）
- 每卡：大数字（1/2/3）+ 条件名 + 说明
- 条件固定：① 同功能 ② 更容易 ③ 马上有效
- 结尾原则：「替代行为不是奖励，而是新的出口。」
- 下方通常接 C37 Replace Grid

---

### C37 Replace Grid

**用途**: 六种来源各自的旧行为 vs 替代行为对照

**规则**:
- 每项：Emoji + 功能名 + 双栏（旧行为红色 ✘ / 替代行为绿色 ✔）
- 旧行为：红色背景，标签「原来的行为」
- 替代行为：绿色背景，标签「可以替代的方式」
- 每栏 3–4 个选项
- 下方附生活例子 + 常见错误（红色小字）
- 适用：展示具体可操作的替代选项

---

### C38 Demand Grid

**用途**: 学校对孩子的多种能力要求

**规则**:
- 3列 grid，每卡：Emoji + 标题 + 要求清单 + 底部注（家/校对比）
- 底部注格式：「家里：[情况]。学校：[情况]。」
- 通常 6 卡（对应学校六项要求）
- 适用：帮家长理解学校环境对孩子的负荷

**P004 U001 六项要求**:
```
👂 感觉处理 / 👥 社交理解 / 📚 注意与学习
🔄 转换 / ⏰ 时间节奏 / ❤️ 情绪控制
```

### C39 Network Box

**用途**: 支持网络的人员结构

**规则**:
- 灰色背景（surface-1）
- 标签：「支持网络里的人」
- 每行：角色名（左，加粗）+ 说明（右）
- border-bottom 分隔
- 适用：帮家长看到支持不是一个人的事

### C40 Support Category

**用途**: 按类别展示学校支持选项，标注门槛

**规则**:
- 每个类别：标题行（Emoji + 名称）+ 2列支持卡 grid
- 绿色卡：标注「今天就能开始」（low-barrier 标识）
- 普通卡：无特别标注
- 每卡：标题 + 说明 + 怎么做（蓝色小字）
- 类别通常 3–4 个
- 适用：帮家长找到门槛最低的切入点

**P004 S002 四类支持**:
```
📢 提前预告和转换支持
👁️ 视觉支持
🌀 感觉和调节支持
📋 任务和学习调整
```

### C41 Observe Grid

**用途**: 家长和老师各自的观察分工

**规则**:
- 双栏，各自独立
- 左栏：家长观察，灰色背景
- 右栏：老师观察，蓝色背景（bg-accent）
- 每栏：标签（谁观察）+ 清单（观察什么）
- 底部说明：「两个观察加在一起，才是孩子一天完整的样子。」
- 适用：建立家校共同观察机制

---

### C42 Understand List

**用途**: 解释某件事对孩子为什么更困难

**规则**:
- 每项：Emoji + 标题 + 说明 + 生活例子（斜体）
- 重点：不是描述障碍，而是帮家长看懂孩子的处境
- 通常 3–5 项
- 结尾：「不是孩子不想…，而是…对他来说代价更高。」
- 适用：社交困难、学习困难、沟通困难等场景

---

### C43 School Box

**用途**: 请学校配合创造机会的清单

**规则**:
- 绿色背景（bg-success）+ 绿色边框
- 标签：「请老师帮忙……」
- 可勾选清单，✓ 标记
- 每条：具体、可操作、不增加大量工作量
- 适用：同伴关系、感觉支持、AAC 在校使用等

---

### C44 Consistency Principles

**用途**: 一致性的层次框架

**规则**:
- 三卡等宽，蓝色背景（bg-accent）
- 每卡：大数字（1/2/3）+ 层次名 + 说明
- 层次固定：① 来源一致 ② 方向一致 ③ 工具一致
- 说明末尾：哪一层最重要 / 哪一层最理想
- 适用：家校一致性框架解释

**Family-School Consistency Framework（全站统一）**:
```
① 来源一致（Understanding）—— 最重要
② 原则一致（Response）
③ 工具一致（Tool）—— 最理想，但不是必须
```

---

### C45 Trigger Grid

**用途**: 多种触发情境的卡片展示

**规则**:
- 2列 grid，每卡：Emoji + 标题 + 说明 + 「→ 可以做」（蓝色小字）
- 每卡简洁，3–4 行
- 通常 4–6 张卡
- 适用：列出「什么情况下需要调整」「什么情况下更难」等多种场景

---

### C46 Setback Tags

**用途**: 可能导致退步的情境标签云

**规则**:
- 标签 pill 样式：小圆角，灰色背景，font-size 13px
- flex-wrap 排列
- 后接说明文字：「这些情况可能让行为/状态暂时变差，不是之前的努力白费了。」
- 结尾：「等状态回来，之前建立的能力通常也会回来。」
- 适用：退步说明，降低家长焦虑

## 组件使用原则（完整版）

**R09 One Page, One Outcome**
每页上线前检查：这页帮家长完成了什么？不是「介绍X」，而是「家长今天能做到X」。

**R10 5–7分钟阅读上限**
超出则拆页。家长在车上、等孩子、晚上11点看。

**R15 Permission Design**
允许告诉家长：可以暂停 / 可以慢一点 / 可以今天什么都不做。

**R16 Reason → Action**
困难处理页：原因 → 今天怎么办。不能只有解释。

**R17 Recovery Path**
Next Step 至少有一个「回到之前 / 重新开始」的选项。

**R24 平台哲学句式**
```
不是 X 的问题，而是 Y。
不是方法错，而是来源不同。
不是情绪问题，而是感觉过载。
不是行为问题，而是转换困难。
```

**R25 先判断，再支持**
所有涉及「孩子为什么这样」的页面，优先提供判断框架，不直接给答案。

## 新路径生产清单（v1.1）

从 P005 开始，每页生产前检查：

```
□ 确认 One Outcome（一句话）
□ 确认 One Moment（一个真实时刻）
□ 选择需要的组件（从本库 C01–C46 选，不新建）
□ Today Box 门槛够低？（R02）
□ Evidence Box 用词正确？（研究发现，不是研究证明）（R22）
□ Professional Block 有「没有也不用担心」？（R06）
□ Next Step 有 Recovery 选项？（R17）
□ Closing 使用固定文字？（R08）
□ 阅读时间 < 7 分钟？（R10）
□ 涉及行为来源：使用 Anori Functional Language？（C30）
□ 涉及家校沟通：使用 Family-School Consistency Framework？（C44）
□ 涉及 ABC 观察：使用标准 ABC Framework 命名？（C32）
□ 组件风格与 P001–P004 一致？
```
