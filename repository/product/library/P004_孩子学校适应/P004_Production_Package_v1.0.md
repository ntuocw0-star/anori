## 1. Project Summary

**User Path**: P004 — 孩子学校适应
**Core Problem**: 孩子在学校适应困难，家长不知道怎么和学校合作，也不确定学校能做什么
**One Sentence**: 学校不是另一个战场，而是支持系统的一部分。而你，是这个系统的中心。
**Primary Audience**: 孩子在学校有适应困难的家长（神经发展差异儿童为主）
**Domain Coverage**: School Participation, Family-School Partnership, Inclusion

**设计原则**:
P004 是 Anori 第一个真正进入「家庭 × 学校共同支持」的 Journey。
目标不是教家长怎么「管」学校，而是帮家长让学校成为支持的一部分。

**核心定位**:
家校合作框架——从「对立」到「共同观察」，从「请求」到「持续合作系统」。

## 3. Information Architecture

```
层级 1  Journey         共情 + 认知重构（学校≠战场）+ 支持网络地图
层级 2  Understanding   学校为什么难（六项要求 + 放学后崩溃）
层级 3  Support         准备什么信息给学校（支持方式 > 诊断）
层级 4  Support+        六个家校合作场景
```

**P004 特有逻辑**:
P004 是 P001–P004 中第一个把「支持网络」作为核心的路径。
Support+ 形成一条从「开口」到「持续系统」的完整链条：
- S001: 开口 → 第一句话，第一次见面
- S002: 支持 → 今天就能开始的选项
- S003: 共识 → 建立共同观察，而不是争对错
- S004: 连结 → 同伴，真实连结 > 社交训练
- S005: 一致 → 方向一致 > 工具复制
- S006: 更新 → 支持是活的，随孩子成长

## 5. One Outcome / One Moment 汇总

**路径核心 One Outcome**:
帮助家长把学校从「需要对抗的地方」，变成「共同支持孩子成长的伙伴」

**路径核心 One Moment**:
第一次主动联系老师，分享一件有效的支持方式

**品牌翻译语言（全路径）**:
- 学校没问题 ≠ 孩子没有压力
- 放学后崩溃 ≠ 在家出问题
- 一致 ≠ 复制，而是方向相同
- 交朋友 ≠ 有连结
- 支持建好了 ≠ 不需要更新
- 分享支持方式 ≠ 贴标签

**Family-School Consistency Framework（P004 原创）**:
```
① 来源一致（Understanding）
② 原则一致（Response）
③ 工具一致（Tool）
```

## 10. UI Pattern & Design Tokens

**P004 新增设计语言**:
- 「今天就能开始」绿色标签：区分低门槛支持
- 支持分类卡：图标 + 标题 + 清单 + 怎么说
- 「老师看到 vs 孩子经历」蓝色对比卡
- 观察信号 Tags（标签云）：轻量视觉，不用列表

**Family-School Consistency Framework 视觉规格**:
- 三卡等宽，蓝色背景（bg-accent）
- 大数字 1/2/3 + 层次名 + 说明
- 依次：Understanding → Response → Tool
