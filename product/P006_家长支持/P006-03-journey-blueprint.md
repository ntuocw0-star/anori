# P006 · 03 — Journey Blueprint
## Caregiver Support / 家长支持

**Sprint 14 · 补档任务**
**上游：** Problem Universe · Problem Cluster v1.0

---

## 核心命题

> 你已经在做一件很难的事。
> 这里不是要告诉你怎么做得更好。
> 而是帮你看见：你需要支持，而且可以去获得支持。

**这不是心理咨询，不是鼓励，不是情绪安慰。**
P006 的工作是帮助家长完成一次归因转移：

```
从："都是我的错，我不够好"
到："我处在一个需要支持的处境里，而且支持是存在的"
```

---

## Journey 的目标

### 用户进入 Journey 时带着什么

家长通常从某一个具体的触发点来——

- "我刚对孩子发了很大的火"
- "我今天又一个人去评估，回来在车里哭了"
- "我已经不知道还能撑多久"
- "家里人又说我想太多了"

他们还没有"照顾者倦怠"这个框架。他们有的是：
- 一个具体的崩溃瞬间或积累已久的疲惫
- 一种"这应该是我能应对的事，但我做不到"的内疚
- 不确定自己的状态是否严重到"值得"寻求帮助

### 用户离开 Journey 时应该知道什么

**认知层（理解框架）：**
1. 照顾者倦怠是真实的，不是脆弱，也不是失败
2. 感到撑不住，不是因为不够爱孩子，而是因为承受了超出个人能力的负担
3. 照顾自己不是等孩子好了以后的事，是现在就必要的事
4. 支持是存在的——专业的、社群的、日常的

**情绪层（归因转移）：**
- 从"我是坏家长"→"我是一个处于高压处境的家长"
- 从"不该抱怨"→"我的感受是真实的，值得被认真对待"
- 从"等我更强了再说"→"我现在就可以去找支持"

**行动层（下一步方向）：**
- 识别自己现在最需要的支持类型（喘息 / 情感连接 / 实际帮助 / 专业支持）
- 知道 P006 里有哪些具体的支持资源
- 知道接受帮助不是失败，而是照顾孩子的一部分

### 用户离开 Journey 时应该消除的误解

| 常见误解 | Journey 给出的理解 |
|---------|-----------------|
| "别的家长都做得到，只有我不行" | 你看到的是别人的外表，不是他们的内心 |
| "是我的错孩子才这样" | 归因是复杂的，但把所有重量压在自己身上不是真相 |
| "我去找帮助是软弱的" | 寻求支持是照顾孩子的一部分，不是放弃 |
| "等孩子稳定了再照顾自己" | 你给孩子的，取决于你自己还剩多少 |
| "我对孩子发火，所以我是坏家长" | 情绪崩溃是系统超载的信号，不是性格的判词 |
| "没有人能理解我" | 有人走过同样的路——他们不一定就在你身边，但他们存在 |

## Journey 页面架构

### Section 1 · 你来到这里，可能是因为某一个很累的时刻

**目标：** 让家长立刻感到"说的就是我"，不需要先解释自己
**内容策略：**
- 列举几个来自 Problem Universe 的具体瞬间（家长语言）
- 不评判，只确认——"你现在的感受，不需要先证明它是真实的"
- 引入核心命题

**核心命题作为转折点：**
> 你已经在做一件很难的事。
> 这里不是要告诉你怎么做得更好。
> 而是帮你看见：你需要支持，而且可以去获得支持。

---

### Section 2 · 撑不住不是失败，是信号

**目标：** 完成第一层归因转移——从"我不够好"到"我在高压处境里"
**内容策略：**
- 解释什么是照顾者倦怠（不用这个词，用描述）
- 说明这不是性格或爱不够多的问题
- 给出一个真实的数据背景（研究发现照顾特殊需求孩子的家长……）

**不说：**
- "你已经做得很好了"（空泛安慰）
- "孩子需要你坚强"（增加压力）
- "一定会好的"（过度承诺）

**说：**
- "感到撑不住，是你的系统在告诉你它需要支持"
- "这不是软弱，这是真实的处境"

---

### Section 3 · 你不是一个人，但孤立感是真实的

**目标：** 承认孤立，同时说明连接是可能的
**内容策略：**
- 承认：这段经历很难被没有走过的人理解
- 说明：孤立感不是因为你不擅长社交，而是因为这个处境本身的特殊性
- 引入：同伴支持（peer support）的价值——不是专业建议，而是被看见

---

### Section 4 · 照顾自己不是以后的事

**目标：** 去除"等孩子好了再说"的延迟逻辑
**内容策略：**
- 直接说：你现在能给孩子的，取决于你自己还剩多少
- 这不是"先照顾好自己才能照顾孩子"的功利逻辑——而是：你的状态本身就值得被照顾
- 引入：照顾自己的最小单位是什么（不需要是度假，可以是今天做一件小事）

---

### Section 5 · 支持是存在的

**目标：** 给出方向，不是空话
**内容策略：**
- 列出四种支持类型（喘息 / 情感连接 / 实际帮助 / 专业支持）
- 说明接下来 P006 里有哪些具体方向
- PathMap 展示 S001–S006

## Cross-path 出口规划

| 出口 | 原因 | 建议位置 |
|------|------|---------|
| P001（沟通）| 家长在使用 AAC / 支持孩子沟通时特别需要自身支持 | s003 · s006 |
| P002（情绪）| 孩子情绪调节困难与家长情绪耗尽互相影响 | s001 · s006 |
| P003（行为）| 行为支持对家长的要求很高，经常是倦怠触发点 | s001 · s006 |
| P004（学校）| 家校协作压力是照顾者倦怠的重要来源 | s003 · s004 |
| P007（感觉）| 感觉敏感孩子的护理特别消耗家长 | s001 · s002 |

**P006 的 Cross-path 特点：**
P006 是所有路径里最需要输入的一条——家长通常先经历了 P001–P005 的某一条，才来到 P006。
因此 P006 的入口应该分布在其他路径的 s005 或 s006 位置——当家长走到路径末尾、感到精力耗尽时，P006 是自然的下一站。

## U001 内容方向（Blueprint）

**标题方向：** 研究怎么说——照顾特殊需求孩子的家长，真实面对着什么

**核心内容：**
1. 照顾者倦怠的定义与发生机制（不用病理化语言）
2. 数据：照顾特殊需求孩子的家长，倦怠率、孤立率、心理健康状况
3. 为什么孤立感如此普遍——结构性原因，不是个人问题
4. 研究发现：同伴支持、喘息服务、专业心理支持的效果
5. Evidence：→ EA-000001/003（照顾者体验研究）

## 写作语调原则（P006 专用）

**始终从家长的感受出发，不从建议出发：**
✅ "你感到撑不住，是真实的"
❌ "你需要学会自我照顾"

**不给空泛安慰，也不给过度承诺：**
✅ "这段路是真的难"
❌ "一定会好的" / "你已经做得很好了"

**归因始终指向处境，不指向性格或能力：**
✅ "你处在一个需要支持的处境里"
❌ "你太敏感了" / "你需要更坚强"

**照顾自己的框架是需求，不是奖励：**
✅ "你的状态本身就值得被照顾"
❌ "先把自己照顾好，才能照顾孩子"

**不用临床语言，不做诊断，不引发恐慌：**
✅ "很多家长走过这里，感觉和你一样"
❌ "这可能是抑郁症的症状"

## One Outcome · One Moment

**One Outcome（用户带走的一件事）：**
> 我需要支持。而且，去寻求支持不是软弱，是我现在需要做的事。

**One Moment（Journey 最重要的一个瞬间）：**
家长读到"你感到撑不住，不是因为你不够好，而是因为你一直在独自承担太多"这句话，停下来，感到被看见。

这个瞬间发生，Journey 就完成了它最重要的工作。
