# Search Clarifier Spec v1.0

**Sprint 15 · Phase 0**
**状态：** 冻结

---

## 核心定义

Search Clarifier 不是"搜不到时的备用方案"。

它是：
> 帮家长把不会表达的问题，变成系统能理解的问题。

---

## 触发条件

以下情况触发 Clarifier：

1. 搜索词命中多个高分 PRB（歧义）
2. 搜索词分数很低但不为零（模糊表达）
3. 搜索词匹配到情绪词而非症状词（如"天天发疯"）
4. 搜索词长度 > 6 字，包含"怎么办"类结尾词

**不触发：**
- 搜索结果已经很清晰（一个高分 PRB 明显领先）
- 搜索词是已知关键词直接命中

---

## 交互流程

```
用户输入: "孩子天天发疯"
        ↓
分析：情绪词 + 行为词，匹配多个 PRB
        ↓
Clarifier 显示：
┌─────────────────────────────────────┐
│  "天天发疯"可能有不同的来源，          │
│  你说的更像哪一种？                   │
│                                     │
│  ○ 情绪崩溃，很难平复                 │
│  ○ 到了某些地方就突然爆发             │
│  ○ 沟通受阻，表达不了需求             │
│  ○ 睡眠不足，白天很烦躁              │
└─────────────────────────────────────┘
        ↓
用户点击"情绪崩溃，很难平复"
        ↓
直接进入 P002/journey 或对应 PRB
```

---

## Clarifier 问题设计原则

**1. 选项用家长语言，不用诊断术语**
✅ "到了某些地方就突然爆发"
❌ "感觉过载导致的崩溃行为"

**2. 选项数量 2–4 个，不超过 4 个**
超过 4 个选项会让家长再次迷失。

**3. 每个选项直接对应一个 PRB 或 Path**
不能点完还是搜索结果，要直接路由。

**4. 提供"都不是，换一种说法"的出口**
不强迫家长选一个不合适的。

## Clarifier 触发词库（初始版）

| 触发词 | Clarifier 类型 | 选项方向 |
|--------|--------------|---------|
| 发疯 / 发脾气 / 崩溃 | 多因素歧义 | 情绪 / 感觉过载 / 沟通受阻 / 疲惫 |
| 打人 / 咬人 | 行为来源歧义 | 沟通受阻 / 感觉寻求 / 情绪失调 |
| 不说话 | 沟通类型歧义 | 没有语言 / 有语言但不用 / 场合性不说 |
| 不去学校 | 原因歧义 | 社交困难 / 感觉环境 / 情绪焦虑 |
| 睡不好 | 睡眠类型歧义 | 入睡困难 / 夜醒 / 睡眠需求不同 |
| 怎么办 / 不知道怎么做 | 通用入口 | 显示主路径选择器 |

## UI 位置

Clarifier 出现在搜索结果页顶部，替换"你搜索的是……"的 banner：

```
当前（无 Clarifier）：
┌─────────────────────────────────────┐
│ 你搜索的是「天天发疯」，下面是可能相关的内容。│
└─────────────────────────────────────┘

加入 Clarifier：
┌─────────────────────────────────────┐
│ "天天发疯"可能来自不同的情况。          │
│ 你说的更像哪一种？                    │
│                                     │
│ [情绪崩溃]  [感觉过载]  [沟通受阻]    │
│ [睡眠不足]  [我不确定，继续搜索]       │
└─────────────────────────────────────┘
```

---

## 与 Question Dictionary 的关系

Clarifier 是实时的 UI 层。
Question Dictionary 是背后的数据层。

```
用户输入 → Clarifier 触发判断（UI）
              ↓
         查询 Question Dictionary（数据）
              ↓
         返回 Clarifier 选项
              ↓
         用户选择 → 直接路由到 PRB/Path
              ↓
         记录：用户原始表达 + 选择结果（学习数据）
```

---

## 实现方案（Sprint 15）

**Phase 1（最小可用）：**
- 在 search.astro 的 doSearch() 里增加 Clarifier 触发判断
- 硬编码初始触发词库（20–30 个词）
- Clarifier 选项点击直接跳转 PRB/Path 页面

**Phase 2（Sprint 16）：**
- Clarifier 数据从 Question Dictionary JSON 动态读取
- 记录 Clarifier 使用数据
